# Implementation Plan: Inactivity Detection

## Approach

The app is a menu-bar tracker whose window is usually hidden, so "inactivity"
means *away from the computer* (system-wide OS idle) тАФ which the WebView cannot
observe. **Rust owns detection.** A tiny FFI to
`CGEventSourceSecondsSinceLastEventType` (CoreGraphics framework, same FFI style
as the existing `control_key_held`, no new crate, no macOS permission prompt)
returns seconds since the last HID input. The existing 1-second tray tick
computes idle while a timer is running and the feature is enabled, updates the
two menu-bar surfaces (icon + title), and emits an `inactivity` event the React
layer subscribes to. This keeps both surfaces in lockstep and immune to WebKit
timer throttling.

The feature is **informational while away** and **actionable on return**. While
idle тЙе threshold: an amber banner under the dropdowns reads `Inactive for {h}h
{m}m`, the tray title becomes `You've been inactive for {h}h {m}m`, and the
tray bear's green status dot turns red (new `tray-inactive.png`). The timer
keeps counting. On return (any input), the tray reverts instantly and the
banner becomes a **persistent** prompt тАФ `You were away {h}h {m}m тАФ
[Discard] [Keep]` тАФ that won't dismiss on its own and blocks Stop until
resolved. **Discard** rewinds the running timer by the away duration (shifts
`startTime` forward, so those minutes never reach AgileDay); **Keep** leaves
them counted. Only the latest away period is discardable.

State is split cleanly: detection + raw idle live in Rust; a small `inactivity`
slice in the existing reducer holds `idleSeconds`, `isAway`, and `pendingReturn`
so the banner and the Stop-guard read one source of truth. Pure helpers
(`formatAway`, discard math, the reducer transition) are unit-tested; the Rust
system call stays thin and untested. The setting reuses the existing
`DisplayPrefs`/`display.json` persistence and the `set_timer_status` push that
already carries `menuBarMode`.

## Data Model

```typescript
// display-store.ts тАФ persisted prefs
interface DisplayPrefs {
  menuBarMode: MenuBarMode;
  theme: ThemeMode;
  inactivityEnabled: boolean; // default false
  inactivityMinutes: number;  // default 10, clamped 1тАУ120
}

// reducer.ts тАФ runtime slice (NOT persisted)
interface InactivityState {
  idleSeconds: number;                       // current system idle; 0 when active
  isAway: boolean;                           // idle >= threshold right now
  pendingReturn: { awaySeconds: number } | null; // set on awayтЖТactive; cleared by Discard/Keep
}

type AppAction =
  | { type: "SET_INACTIVITY"; payload: { idleSeconds: number; isAway: boolean } }
  | { type: "RESOLVE_RETURN" }
  // тАжexisting actions
```

`SET_INACTIVITY` transition rule (in reducer): when `prev.isAway === true` and
`payload.isAway === false` and `prev.idleSeconds > 0`, set
`pendingReturn = { awaySeconds: prev.idleSeconds }` (the last away value, minute
granularity). `RESOLVE_RETURN` clears `pendingReturn`. Discard dispatches
`SET_TIMER { startTime: computeDiscardStartTime(...) }` then `RESOLVE_RETURN`.

```rust
// lib.rs тАФ Rust тЖТ JS event payload (emitted on change of (isAway, idle_minute))
#[derive(Clone, serde::Serialize)]
struct InactivityPayload { idle_seconds: u64, is_away: bool }

// FFI (no new crate; link CoreGraphics framework)
#[cfg(target_os = "macos")]
fn system_idle_seconds() -> u64 {
    #[link(name = "CoreGraphics", kind = "framework")]
    extern "C" { fn CGEventSourceSecondsSinceLastEventType(state: i32, ev: u32) -> f64; }
    const HID_STATE: i32 = 1;        // kCGEventSourceStateHIDSystemState
    const ANY_EVENT: u32 = 0xFFFF_FFFF; // kCGAnyInputEventType
    (unsafe { CGEventSourceSecondsSinceLastEventType(HID_STATE, ANY_EVENT) }).max(0.0) as u64
}
```

## Risks & Edge Cases

- **CoreGraphics FFI linking** [decided] тАФ use `#[link(name = "CoreGraphics", kind = "framework")]`; verify it compiles as the first step of task 6. Fallback: pull `core-graphics` crate only if the build rejects it.
- **Blocking Stop while a prompt is pending** тАФ if the tray Stop is hit while the window is hidden, the block is invisible. Mitigation: a return prompt only arises *after* activity (window typically open); document the behavior. (Chosen per the grill's "persistent prompt" option.)
- **`awaySeconds` accuracy** [decided: minute-granular] тАФ system idle resets to 0 on the return input, so the active event carries idle 0; we freeze `prev.idleSeconds` (minute granularity, тЙд60s slack). Acceptable for a minutes-level feature; AgileDay logs minutes and finalize rounds to 15-min.
- **Disable mid-prompt** [decided] тАФ disabling the toggle (or the timer stopping) while a return prompt is unresolved clears it, treated as Keep (tasks 4, 8, 10).
- **Light-amber banner in dark mode** [decided] тАФ reuses the existing hardcoded `bg-amber-50`/`amber-700` style of the empty-description warning for consistency; stays light in dark mode by design.
- **Event spam** тАФ emit `inactivity` only when `(is_away, idle_minute)` changes, not every second.
- **Reading without input counts as idle** тАФ inherent to system idle; the user-set threshold is the mitigation.
- **Branch dependency** тАФ the setting sits under the Appearance card, which only exists on `feature/dark-mode`; this branch is cut from it. Merge order matters.
